Положите сюда изображения, используемые в портфоліо или загрузите их на CDN/облако.

- `avatar.jpg` — ваша фотография. Рекомендуемый размер: квадрат ~400×400 (можно больше — браузер уменьшит).
- `work-kazatska.jpg` — скриншот проекта/превью для секции "Мої роботи". Рекомендуемый размер: ~1200×600.

Форматы: JPEG, WebP или PNG.

Как вынести на облако и подключить:
1. Загрузите файлы из папки `images/` в ваш CDN или облачное хранилище (S3, Cloudflare R2, GitHub Pages, Netlify, Vercel и т.д.).
2. Получите базовый URL, где лежат файлы. Для GitHub Pages это обычно `https://<user>.github.io/<repo>/`.
3. Откройте `index.html` и/или `resume.html` и в теге `<head>` заполните мета-поле `assets-base`.

	 - Если вы загрузили файлы в корень сайта (т.е. `https://.../<repo>/images/...`), укажите базу как корень сайта (пример для GitHub Pages):

		 <meta name="assets-base" content="https://<user>.github.io/<repo>/">

		 Скрипт автоматически добавит `images/...` к этому базовому URL.

	 - Если вы загрузили файлы прямо в каталог `images/` на CDN и хотите подставлять полный путь, можно указать базу с `images/` в конце:

		 <meta name="assets-base" content="https://cdn.example.com/portfolio/images/">

4. Обновите страницу — все локальные `src="images/..."` будут подставлены относительно `assets-base`.

Примечание: если `assets-base` пустой (по умолчанию), локальные пути `images/...` остаются как есть и будут загружаться из того же хоста, где размещена страница (подходит для GitHub Pages, если вы задеплоили `images/` в репозиторий).
